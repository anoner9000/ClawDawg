# Agent Onboarding Checklist

## Identity
- [ ] SOUL.md created and reviewed
- [ ] Role defined (one purpose, no overlap)
- [ ] Default message formats confirmed

## Access & permissions
- [ ] ACCESS.md created for relevant projects
- [ ] Tool access scoped (least privilege)
- [ ] Execution disabled unless explicitly intended

## Team integration
- [ ] Added to agent registry (AGENTS.md)
- [ ] Announced in team bus (type=UPDATE, summary=New agent onboarded)
- [ ] Assigned initial L1 tasks (read-only)

## Governance
- [ ] Level set (L1 default)
- [ ] Promotion criteria written
- [ ] Review cadence scheduled (monthly)

## First assignment (trial)
- [ ] Low-risk task with clear acceptance criteria
- [ ] Auditor verifies output quality
- [ ] Deiphobe closes task and records outcome
